# Акча табууну үйрөтүүчү Telegram бот

Бул бот:
- Тил тандоону колдойт (Кыргызча / Орусча)
- Боттун ичине кирүү үчүн 3 каналга катталууну милдеттүү кылат
- Ар бир кирүүдө (жана ар бир бөлүмгө өтөрдө) катталуусун кайра текшерет —
  эгер колдонуучу каналдан чыгып кетсе, "катталган эмессиз" деп кайра эскертет
- Курстар бөлүмүн камтыйт (сабактар тизмеси → толук түшүндүрмө текст + YouTube видео шилтемеси)
- Админ панелин камтыйт: бардык колдонуучуларга текст/фото/видео жөнөтүү (рассылка), колдонуучулар санын көрүү
- 24/7 колдоо баскычын камтыйт

## 1. Орнотуу

```bash
python3 -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

## 2. Жөндөө (config.py)

`config.py` файлын ачып төмөнкүлөрдү толтуруңуз:

1. **BOT_TOKEN** — Telegram'да `@BotFather`'га жазып, "/newbot" менен жаңы бот түзүп,
   ал берген токенди жабыштырыңыз.

2. **ADMIN_IDS** — Сиздин (же башка админдердин) Telegram ID'иңиз.
   ID'иңизди билбесеңиз, `@userinfobot` ботуна жазыңыз — ал ID'иңизди көрсөтөт.
   Бир нече админ кылса болот: `ADMIN_IDS = [111111, 222222]`

3. **CHANNELS** — Катталуу милдеттүү болгон 3 канал.
   - `chat_id`: `@kanal_username` түрүндө же сандык ID (`-1001234567890`)
   - `url`: колдонуучу басып өтүүчү шилтеме
   - **ЭСКЕРТҮҮ:** бот ошол 3 каналдын **админи** болушу керек, болбосо
     Telegram API катталганын текшерүүгө уруксат бербейт!

4. **SUPPORT_CONTACT** — колдоо кызматынын @username же телефону.

## 3. Курстардын мазмунун өзгөртүү (courses_data.py)

`courses_data.py` файлында `COURSES` сөздүгүндө `ky` жана `ru` тилдери үчүн
сабактар тизмеси турат. Ар бир сабакта:
- `title` — сабактын аталышы
- `text` — толук түшүндүрмө текст
- `youtube` — ошол сабакка тиешелүү YouTube видеонун шилтемеси

Жаңы сабак кошуу үчүн эки тилге да (ky жана ru) бирдей орунда кошуңуз
(тизмелердин узундугу бирдей болушу керек).

## 4. Ботту иштетүү

```bash
python bot.py
```

Бот туура жөндөлсө, терминалда "Бот иштеп баштады..." деген билдирүү чыгат.

## 5. Кантип иштейт (кыскача)

- `/start` → тил тандоо (биринчи жолу) → 3 каналга катталганы текшерилет
- Катталбаган болсо → каналдардын шилтемелери + "✅ Текшерүү" баскычы чыгат
- Катталган болсо → башкы меню: 📚 Курстар, 🆘 Колдоо, 🌐 Тил, (админ болсо) 🛠 Админ панель
- Курстар бөлүмүнө же башка бөлүмгө кирээрде да катталуу кайра текшерилет
- Админ панелинде:
  - **📢 Бардыгына билдирүү жөнөтүү** — баскычты басып, кийинки билдирүүңүздү
    (текст, фото, видео — каалаганы) жөнөтсөңүз, ал бардык колдонуучуларга
    автоматтык түрдө жөнөтүлөт. Жокко чыгаруу: `/cancel`
  - **📊 Колдонуучулар саны** — ботту колдонгондордун жалпы санын көрсөтөт

## 6. БЕКЕР 24/7 интернетке чыгаруу: Render + UptimeRobot

Render'дин **Background Worker** тарифи акысыз эмес ($7/айдан баштап).
Бирок **Web Service** тарифи бекер — бот ушул схема менен иштейт:

- `bot.py` эми Telegram polling'ге кошумча кичинекей HTTP сервер да ачат
  (`/` жана `/health` жолдору). Render аны "веб сайт" катары көрөт жана
  бекер тейлейт.
- Кемчилиги: Render'дин бекер Web Service'и **15 мүнөт эч ким кирбесе
  уктап калат**. UptimeRobot (бекер сервис) аркылуу ар 5-10 мүнөт сайын
  ботко "ping" жибериптирип, аны дайыма ойгоо кармайбыз.
- Маалымат базасы (SQLite, `bot.db`) бот уктап-ойгонгондо да сакталат
  (Render'дин дискинде калат), бирок **жаңы deploy (код өзгөртүп
  кайра жайгаштырганда) base жоголушу мүмкүн** — өндүрүштүк колдонууда
  көп колдонуучу чогулса, кийинчерээк акылуу диск же тышкы базага
  (мис. Render Postgres) өтүү сунушталат.

### 6.1. GitHub'ка жайгаштыруу

```bash
cd moneybot
git init
git add .
git commit -m "first commit"
```

GitHub'да жаңы repo түзүп (мисалы `moneybot`, **private** кылып):

```bash
git remote add origin https://github.com/SIZDIN_USERNAME/moneybot.git
git branch -M main
git push -u origin main
```

`.gitignore` болгондуктан `.env` жана `bot.db` GitHub'ка эч качан кетпейт.

### 6.2. Render'де Web Service түзүү

1. [render.com](https://render.com) дан каттал (GitHub менен кирсе ыңгайлуу)
2. **"New +"** → **"Web Service"** (Background Worker эмес!)
3. GitHub repo'ңузду тандаңыз (`moneybot`)
4. Толтуруу:
   - **Name**: каалаганыңыз (мис. `moneybot`)
   - **Runtime**: `Python 3`
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python bot.py`
   - **Instance Type**: `Free`
5. **"Environment"** бөлүмүнө сыр маалыматтарды кошуу:

| Key | Value |
|---|---|
| `BOT_TOKEN` | BotFather'дан алган чыныгы токен |
| `ADMIN_IDS` | сиздин Telegram ID (`111111111`) |
| `SUPPORT_CONTACT` | `@your_support_username` |

6. **"Create Web Service"** басыңыз

Render код тартып алып, орнотуп, иштетет. Логдордо
`"Бот иштеп баштады..."` жана `"Веб-сервер ... портто ачылды"` көрсө —
даяр. Жогору жагында сизге берилген сайттын дареги турат
(мис. `https://moneybot-xxxx.onrender.com`) — аны кийинки кадамда колдонобуз.

### 6.3. UptimeRobot менен ойготуп туруу (бекер)

1. [uptimerobot.com](https://uptimerobot.com) дан бекер каттал
2. **"+ New Monitor"** басыңыз
3. Толтуруу:
   - **Monitor Type**: `HTTP(s)`
   - **Friendly Name**: `MoneyBot`
   - **URL**: Render берген дарек + `/health`
     (мис. `https://moneybot-xxxx.onrender.com/health`)
   - **Monitoring Interval**: `5 minutes` (бекер тарифте эң кичинекей убакыт)
4. Сактаңыз

Эми UptimeRobot ар 5 мүнөт сайын ботко кирип, аны "уктатпай" кармайт.
Бот дайыма иштеп турат.

### Альтернатива: VPS (туруктуу серверге өтсөңүз)

Эгер кийинчерээк туруктуу серверге (VPS) өтсөңүз, ботту systemd
кызматы катары иштетсе болот:

```ini
[Unit]
Description=MoneyBot Telegram Bot
After=network.target

[Service]
WorkingDirectory=/path/to/moneybot
ExecStart=/path/to/moneybot/venv/bin/python bot.py
Restart=always

[Install]
WantedBy=multi-user.target
```

VPS'те HTTP сервер/UptimeRobot'тун кереги жок, ал жерде бот үзгүлтүксүз
иштейт.

## Файлдардын структурасы

```
moneybot/
├── bot.py              # Ботту иштетүүчү негизги файл
├── config.py            # Токен, админдер, каналдар, контакт
├── database.py           # SQLite база (колдонуучулар, тилдер)
├── locales.py            # Бардык тексттер (ky/ru)
├── courses_data.py        # Сабактардын мазмуну
├── keyboards.py           # Inline баскычтар
├── subscription.py         # Катталууну текшерүү логикасы
├── requirements.txt
└── handlers/
    ├── start.py           # /start, тил тандоо, башкы меню
    ├── courses.py          # Сабактар бөлүмү
    ├── support.py          # 24/7 колдоо
    └── admin.py            # Админ панели, рассылка
```
